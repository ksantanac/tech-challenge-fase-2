Kubernetes na AWS (EKS) · 5 microsserviços · serviços gerenciados AWS

Como ler o diagrama

  • Setas cinzas: fluxo síncrono (requisição HTTP do usuário).
  • Setas laranjas tracejadas: fluxo assíncrono via fila.
  • Chamadas internas: flag e targeting validam a chave no auth-service.
  • evaluation-service consulta flag + targeting e cacheia o resultado no Redis.
  • evaluation publica um evento no SQS a cada avaliação.
  • analytics consome o SQS e grava os eventos no DynamoDB.
